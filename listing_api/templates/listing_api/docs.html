API Docs

REST API for automatic garment analysis. Multipart image upload → structured JSON.

Overview

Base URL: https://theoracle.up.railway.app/listing
Content-Type: multipart/form-data
Response: application/json

One endpoint — POST /api/v1/analyze/ — takes 1–6 photos of a single garment and returns structured metadata, SEO listing copy, and pre-computed embeddings. Typical latency: 10–20s for 3 images with embeddings included.

Authentication

All API requests require an API key. Send it in either header:

Authorization: Bearer sk_live_...
# or
X-API-Key: sk_live_...
API keys are currently issued manually during our pilot phase. Email eumiejhong@gmail.com to request one.

Rate limits

Limits are per-key and configurable. Defaults: 30 requests/minute, 5,000 requests/day. Exceeding them returns 429 Too Many Requests with a Retry-After header.

POST /api/v1/analyze/

Analyze 1–6 photos of a single garment.

Request — multipart form fields

FieldTypeRequiredDescription
imagesfile[]Yes1–6 image files. JPEG, PNG, or HEIC. ≤10MB each.
rolesstring (CSV)NoComma-separated roles matching images order. Allowed: {{ image_roles|join:", " }}. Defaults: first image front, rest detail.
hintstringNoFree-text hint (e.g. "Saint Laurent wool coat, IT 40"). Used as soft guidance; may be overridden if the image disagrees.
include_embeddingsboolNoDefault true. Set false to skip BGE + CLIP embeddings (saves ~1s).
include_copyboolNoDefault true. Set false to skip listing copy generation.

Response — 200 OK

See Response schema below for all fields.

GET /api/v1/health/

Embeddings usage guide

Both vectors are L2-normalized, so cosine similarity reduces to a dot product.

Text embedding

  • Model: BAAI/bge-base-en-v1.5
  • Dim: 768
  • What's embedded: style_descriptors + generated listing description
  • Use for: "find similar listings" text queries, clustering by aesthetic, deduplication

Image embedding

  • Model: clip-ViT-B-32 (OpenAI CLIP via sentence-transformers)
  • Dim: 512
  • What's embedded: the front image (or first image if no role given)
  • Use for: visual similarity search, "more like this", customer-photo matching

Storage

Store as float4[] in Postgres with pgvector, or push to Pinecone/Weaviate. Example (pgvector):

CREATE TABLE listings (
  id           bigserial PRIMARY KEY,
  metadata     jsonb NOT NULL,
  text_vec     vector(768),
  image_vec    vector(512)
);
CREATE INDEX ON listings USING ivfflat (text_vec  vector_cosine_ops);
CREATE INDEX ON listings USING ivfflat (image_vec vector_cosine_ops);

curl

curl -X POST https://theoracle.up.railway.app/listing/api/v1/analyze/ \
  -H "Authorization: Bearer sk_live_..." \
  -F "images=@front.jpg" \
  -F "images=@label.jpg" \
  -F "images=@detail.jpg" \
  -F "roles=front,label,detail" \
  -F "hint=Saint Laurent wool coat" \
  -F "include_embeddings=true"

Python

import requests

r = requests.post(
    "https://theoracle.up.railway.app/listing/api/v1/analyze/",
    headers={"Authorization": "Bearer sk_live_..."},
    files=[
        ("images", ("front.jpg",  open("front.jpg",  "rb"), "image/jpeg")),
        ("images", ("label.jpg",  open("label.jpg",  "rb"), "image/jpeg")),
        ("images", ("detail.jpg", open("detail.jpg", "rb"), "image/jpeg")),
    ],
    data={
        "roles": "front,label,detail",
        "hint": "Saint Laurent wool coat",
        "include_embeddings": "true",
    },
    timeout=60,
)
r.raise_for_status()
result = r.json()

print(result["metadata"]["brand"], "/", result["metadata"]["subcategory"])
print(result["listing"]["title"])

Node.js

import fs from "node:fs";
import FormData from "form-data";
import fetch from "node-fetch";

const form = new FormData();
form.append("images", fs.createReadStream("front.jpg"));
form.append("images", fs.createReadStream("label.jpg"));
form.append("images", fs.createReadStream("detail.jpg"));
form.append("roles", "front,label,detail");
form.append("hint", "Saint Laurent wool coat");

const res = await fetch(
  "https://theoracle.up.railway.app/listing/api/v1/analyze/",
  {
    method: "POST",
    headers: { Authorization: "Bearer sk_live_..." },
    body: form,
  },
);
const data = await res.json();
console.log(data.metadata.brand, "/", data.metadata.subcategory);
